POST/OrderCMI/v2/summary/get

ข้อควรรู้

  • ดึงข้อมูลคำสั่งซื้อ พ.ร.บ. (CMI) พร้อมคำสั่งซื้อภาคสมัครใจ (VMI) ที่ผูกกัน และข้อมูลการชำระเงิน (Payment) ในการเรียกครั้งเดียว
  • รับเฉพาะ CMI OrderNo เท่านั้น — ระบบจะ resolve VMI OrderNo เองจาก cmi_order.RelationOrderNo
  • Payment เป็น record เดียวที่ใช้ร่วมกันระหว่าง CMI/VMI ส่วน PaymentTransaction จะถูกโหลดแยกต่างหากในแต่ละ order (CMIOrder.PaymentTransaction / VMIOrder.PaymentTransaction)
  • ถ้า OrderNo ไม่มีอยู่ในระบบจะได้ Status 400, ErrorCode 11033404 ("Order does not exist")
  • ใช้สำหรับหน้าสรุปคำสั่งซื้อ/แก้ไขคำสั่งซื้อกรณีงานยกเลิก เพื่อนำข้อมูลเดิมมาแสดงในฟอร์ม (เกี่ยวข้องกับ RFC-20260623 BUG-2)
