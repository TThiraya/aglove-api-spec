POST/Challenge/list

ข้อควรรู้

  • ใช้ดึงรายการ Challenge ทั้งหมดที่ Agent (เจ้าของ Token) มองเห็น พร้อมสรุปจำนวน Quest/ความคืบหน้า (TotalQuestCount, CompleteQuestCount) และรางวัลสูงสุดของ Challenge (RewardGrant)
  • AGNEW เรียก endpoint นี้เพื่อแสดงรายการ Challenge ทั้งหมดในหน้า Challenge (Agent App) — ใช้คู่กับ /Challenge/quest/list เพื่อดึงรายละเอียด Quest ของแต่ละ Challenge
  • Authorization header (Bearer ) ต้องตรงกับ Token ที่ส่งใน body — ถ้าไม่ส่ง header จะได้ ErrorCode 90000991 'Authorization header is missing or empty.'
  • CR: เปลี่ยน logic การแสดงผล Challenge จากเดิมใช้ flag IsActive มาใช้ field PublishStartDate/PublishEndDate แทน — ถ้าทั้งสอง field เป็น null จะแสดง Challenge เสมอ ถ้ากำหนดวันที่ จะแสดงเฉพาะช่วง PublishStartDate ≤ วันที่ปัจจุบัน ≤ PublishEndDate เท่านั้น (ดู Issue #74)
  • ⚠️ ข้อควรระวัง: ต้องตั้ง PublishEndDate ให้ไกลกว่าวันที่ EndDate ของ Challenge จริง เผื่อเวลาให้ Agent กดรับรางวัลหลังกิจกรรมจบ ไม่เช่นนั้น Challenge จะหายไปจากรายการก่อนกดรับรางวัลได้
