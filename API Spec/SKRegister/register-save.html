POST/SKRegister/register/save

ข้อควรรู้

  • บันทึกข้อมูลที่แก้ไขของ Order กลับเข้าระบบ (ฝั่งผู้สมัคร) หลังจากดึงข้อมูลด้วย /register/summary/get
  • ฟิลด์ที่เป็น GUID (PrefixID, ProvinceID, DistrictID, SubDistrictID ฯลฯ) ต้อง resolve จาก Master API ก่อนเสมอ ห้ามกรอกเอง
  • ระบบจะสร้าง DefaultAddressID/DeliveryAddressID ใหม่ทุกครั้งที่ save (ไม่ใช่ update record เดิม) และ derive ZipCode จาก SubDistrictID อัตโนมัติ
  • Payment.* และ PAInsuranceName ห้ามแก้ไข เพราะผูกกับการชำระเงินที่สำเร็จแล้ว
