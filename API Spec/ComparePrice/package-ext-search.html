ข้อควรรู้

  • ใช้ค้นหาแพ็กเกจประกันภัยรถยนต์ (VMI/CMI) ตามเงื่อนไข CarBrand/CarModel/CarCost/ClassIDS/CompanyCode เพื่อเปรียบเทียบราคา — ดู field logic ของ ClassIDS ↔ CarCost ใน memory aglove_compareprice_ext_search
  • Request ต้องใช้ PascalCase และ ID ส่วนใหญ่เป็น string (ยกเว้น CarCost เป็น number) — ผิด schema จะได้ ErrorCode B261999
  • field `IsFakeDeduct` (อยู่ใน PackageResult) = แพ็กเกจนี้มีค่า Deduct (ความรับผิดส่วนแรก) กำหนดไว้ แต่มีการอนุโลมไม่เก็บจริงจากลูกค้า — FE ต้องแสดงข้อความ/badge แจ้งเตือนผู้ใช้เมื่อ IsFakeDeduct=true (ดู Issue #TBD)
  • PackageResultCMI จะเป็น null เสมอสำหรับการค้นหาแบบ VMI (ประกันภาคสมัครใจ) — ดู memory compareprice_cmi_flow เรื่องความแตกต่าง CMI/VMI
