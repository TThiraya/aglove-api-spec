POST/SKRegisterBackEnd/register/save

ข้อควรรู้

  • Admin-edits ข้อมูล registration order: ข้อมูลส่วนตัว, คู่สมรส, ผู้รับประโยชน์, PA package, บัญชีธนาคาร, ที่อยู่, และไฟล์ต่างๆ
  • เรียกซ้ำไม่ได้หลัง IsApprove=true (CC02206)
  • DefaultAddress/DeliveryAddress: ส่ง null เพื่อไม่แตะค่าเดิม — ถ้าส่งมาจะ physically delete row เดิมและสร้างใหม่เสมอ
  • FileID ส่ง = no-op (ไม่ clear ค่าเดิม); ส่ง Temp FileID ใหม่เมื่อต้องการเปลี่ยนเท่านั้น
  • EKYCFaceFileID bug: publish ใช้ EKYCDocumentPublish แทน — workaround คือเปลี่ยนทีละ field
  • IDCard และ OccupationID: validate แต่ไม่ persist — response คืนค่าจาก DB เดิม
