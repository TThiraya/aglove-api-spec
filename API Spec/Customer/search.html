POST/Customer/search

ข้อควรรู้

  • ค้นหาลูกค้าแบบไม่ระบุ Owner (ต่างจาก /Customer/grid/customer/by-owner/list ที่กรองเฉพาะลูกค้าของ Owner เดียว) — คืนทุกรายการที่ IDCard ตรงกัน แม้จะเป็นคนละ CustomerID/Owner
  • SearchType รองรับอย่างน้อย 'IDCard' (ค้นด้วยเลขบัตรประชาชน/เลขทะเบียนนิติบุคคล)
  • โครงสร้าง response เป็น envelope กลาง (Status/Message/ErrorCode/ErrorMessage/Data/Options) ต่างจาก grid endpoint ที่เป็น DataTables (draw/recordsTotal/...)
  • CustomerType: I = นิติบุคคล (บริษัท), P = บุคคลธรรมดา
