ข้อควรรู้

  • ใช้แก้ไขข้อมูล Order ที่สร้างไว้แล้ว (คู่กับ /OrderGEI/create) ก่อนเข้าสู่ขั้นตอนชำระเงิน เช่น flow แจ้งงานต่างด้าว (PA Foreigner)
  • ตัวอย่างนี้คือ request/response จริงที่ "ทำรายการสำเร็จ" (Status 200) — เคสรับกรมธรรม์อิเล็กทรอนิกส์ (PDF/อีเมล) แต่ใบกำกับภาษียังจัดส่งจริงทาง DHL
  • AddressID ในตัวอย่างนี้เป็นค่าจริงที่ระบบสร้างไว้ก่อนแล้วจากขั้นตอน /OrderGEI/create ของ OrderNo เดียวกัน — เรียก /OrderGEI/save ครั้งถัดไปต้องใช้ AddressID ที่มีอยู่จริง ห้ามส่งค่าว่าง (ไม่เช่นนั้นจะได้ error 9002102 'Default AddressID cannot be empty')
  • Customer.TaxInvoiceDeliveryAddress.ReferenceType ในตัวอย่างนี้ยังเป็นค่า "OrderNo" (ไม่ถูกต้องตามที่ควรจะเป็น) แต่ไม่ error เพราะ IsTaxInvoiceDeliveryAddressSameAsDefault = true ทำให้ backend ไม่ตรวจสอบ object นี้ — ถ้าตั้ง false ต้องแก้ ReferenceType เป็น "TAXINVOICE_DELIVERY" ไม่เช่นนั้นจะได้ error 9002810 (ดูรายละเอียด bug ที่พบจริงใน AGNEW Issue #56)
  • พบ bug ที่ FE: เมื่อผู้ใช้เปลี่ยนช่องทางรับกรมธรรม์เป็น PDF ระบบเคยรีเซ็ต DeliveryMethod2 (ใบกำกับภาษี) เป็น EXCLUDE/ELECTRONIC ผิดไปด้วย ที่ถูกต้องใบกำกับภาษีต้องคงเป็น TAXINVOICE/DELIVERY/DHL เสมอในเคสนี้ (ดู AGNEW Issue #56)
