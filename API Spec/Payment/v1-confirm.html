POST/Payment/v1/confirm

ข้อควรรู้

  • Confirm เพื่อ generate Invoice No และทำ final validation สำหรับคำสั่งซื้อแบบ DUAL (CMI + VMI) — ใช้แทน endpoint เดิม /Payment/confirm (schema เก่า ใช้ OrderNo ตัวเดียว)
  • รับเฉพาะ CMIOrderNo/VMIOrderNo แบบแยกกัน — ถ้าส่ง schema เก่า (OrderNo) จะได้ error "Both CMI Order No and VMI Order No cannot be empty" (ErrorCode 70041103)
  • ถ้า Order ถูก confirm ไปแล้ว จะได้ Status 400, ErrorCode 70041807 ("Payment was success.") — เรียกซ้ำไม่ได้กับ order เดียวกัน
  • BUG ที่พบจากการ migrate: ฝั่ง Frontend (summary.vue:344) อ่าน PaymentType.toLowerCase() โดยไม่ป้องกัน null — ถ้า response คืน PaymentType เป็น null (พบในเคสหักส่วนลดบางส่วน) จะเกิด runtime TypeError ทำให้ UI แสดง Error popup ทั้งที่ HTTP ตอบ 200 และ PaymentStatus: Success (ดู RFC-20260623 BUG-1 / 724-Market/AGNEW#49)
  • ค่าส่วนลด (CMIDiscountValue/VMIDiscountValue) ต้อง round เป็นทศนิยม 2 ตำแหน่งก่อนส่งเข้า /Payment/v1/save ที่มาก่อนหน้านี้ ไม่เช่นนั้นจะหลุดเป็นทศนิยมยาวผิดปกติ (ดู BUG-1 เดิมใน #44)
